# ROADMAP.md Template (Version 1)

Use this structure for roadmap files generated by the Phase 7 bootstrap work. The human-readable sections mirror the machine-readable JSON payload so downstream tools (card materializer, architecture generator, validators) stay in sync.

```
---
roadmap_version: 1
project_goal: <one-line project goal>
generated_at: <YYYY-MM-DD>
---

# Roadmap

Summary:
<project goal and any high-level notes>

Phase Overview:
- Phase <n> — <label> (<current|planned|done>)
- ...

## Phase <n> — <label> (<status>)

Summary:
Short description of the phase scope.

Tasks:
- [ ] <code> <title> — <summary>
  - Owner: <optional>
  - Risk: <optional>
  - Acceptance Criteria:
    - [ ] <criterion>
  - Status: <backlog|in-progress|done>
  - Parallelizable: true|false

Roadmap (machine readable):
```json
{
  "version": 1,
  "projectGoal": "<goal>",
  "generatedAt": "<YYYY-MM-DD>",
  "manualNotes": "<optional>",
  "phases": [
    {
      "number": 7,
      "label": "project-bootstrap",
      "status": "planned",
      "summary": "Derived from project tasks.",
      "tasks": [
        {
          "code": "7.1",
          "title": "ROADMAP Spec & Generator",
          "summary": "Define schema and generator.",
          "owner": "bri",
          "risk": "normal",
          "status": "backlog",
          "acceptanceCriteria": ["Specify schema", "Provide template", "Add generator tests"],
          "parallelizable": false
        }
      ]
    }
  ]
}
```

Manual Notes:
Optional freeform notes retained on regeneration.

History:
- <YYYY-MM-DD>: Roadmap generated or regenerated.
```

Regeneration Guardrails
- Keep `roadmap_version` at the top for migrations.
- Preserve the `Roadmap (machine readable)` JSON block; validators read this first.
- Manual Notes and History are preserved when regenerating with the generator.
